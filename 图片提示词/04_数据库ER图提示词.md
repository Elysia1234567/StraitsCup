# 同源 OmniSource 数据库 ER 图提示词

## 统一风格基准

ER 图要和前面视觉保持一致：深蓝数字科技风、半透明玻璃卡片、蓝紫发光线条、少量金色强调主键与外键。图形必须清晰、可读、专业，适合放进作品设计说明书或答辩 PPT。不要做成普通黑白数据库截图，也不要让装饰影响字段阅读。

## 1. 核心业务数据库 ER 图

**用途**：数据库设计、数据结构设计章节。

**提示词**：
生成一张专业清晰的数据库 ER 图，主题为“同源 OmniSource 非遗多智能体平台核心数据库”，深蓝数字科技风背景，表结构使用半透明玻璃卡片，主键字段用金色钥匙图标或金色标记，外键关系用蓝紫发光连线，整体 16:9 横版，字段文字必须清晰可读。

包含以下 6 张表及字段：

1. agent（Agent 预设角色表）
字段：id PK, agent_code UK, name, avatar, role_type, personality, prompt_template, knowledge_scope, language_style, constraints, max_tokens, temperature, top_p, is_preset, sort_order, status, create_time, update_time, is_deleted

2. cultural_theme（非遗文化主题表）
字段：id PK, theme_code UK, name, description, category, cover_image, region, era, knowledge_base, status, create_time, update_time, is_deleted

3. chat_room（聊天室表）
字段：id PK, room_code UK, user_id, theme_id FK, name, description, max_members, member_count, message_count, status, create_time, update_time, is_deleted

4. chat_room_member（聊天室成员表）
字段：id PK, room_id FK, member_type, user_id, agent_id FK, display_name, avatar, role_in_room, last_speak_time, speak_count, join_time, status, create_time, update_time, is_deleted

5. chat_message（聊天消息表）
字段：id PK, room_id FK, message_type, sender_type, sender_id, sender_name, sender_avatar, content, image_url, reply_to_message_id FK, metadata, is_stream, stream_id, search_enabled, search_results, feedback_status, feedback_time, create_time, update_time, is_deleted

6. image_generation_task（文生图任务表）
字段：id PK, task_id UK, user_id, room_id FK, prompt, style, status, result_url, error_message, progress, model, create_time, update_time

关系要求：
chat_room.theme_id -> cultural_theme.id，表示一个非遗主题可对应多个聊天室；
chat_room_member.room_id -> chat_room.id，表示一个聊天室有多个成员；
chat_room_member.agent_id -> agent.id，表示 Agent 可作为聊天室成员；
chat_message.room_id -> chat_room.id，表示一个聊天室有多条消息；
chat_message.reply_to_message_id -> chat_message.id，表示消息可回复消息；
image_generation_task.room_id -> chat_room.id，表示生图任务可关联聊天室；
image_generation_task 生成完成后结果可通过 chat_message.image_url 回流到聊天记录，用虚线关系表达。

布局要求：
cultural_theme 放左上，agent 放左下，chat_room 放中间，chat_room_member 放中下，chat_message 放右中，image_generation_task 放右下；关系线尽量不交叉；每张表标题醒目，字段分组清楚；整体留有边距，适合说明书阅读。

**反向提示词**：
字段乱码，文字不可读，表太小，线条交叉严重，装饰遮挡字段，普通黑白截图风，表数量缺失，错误关系。

## 2. 数据流辅助图：多存储协同架构

**用途**：补充说明 MySQL、Redis、Milvus、OSS 的分层存储关系。

**提示词**：
生成一张“同源 OmniSource 多存储协同数据流图”，深蓝科技风，中央是 Spring Boot 业务编排层，左侧是 Vue 3 Web 端和 UniApp 移动端，右侧是第三方 AI 服务；底部四个存储模块：MySQL 保存聊天室、消息、Agent、非遗主题、生图任务；Redis 保存会话状态、流式回复、任务进度、热点缓存；Milvus 保存非遗资料语义向量并支持 RAG 检索；阿里云 OSS 保存头像、生成图片、非遗展示图和资源文件。用蓝紫数据流箭头连接：前端请求进入后端，后端读写 MySQL，短期状态进入 Redis，知识检索进入 Milvus，图片文件进入 OSS，AI 生成结果返回聊天室消息。16:9，半透明玻璃面板，发光箭头，少量金色强调核心数据，文字清晰可读。

**反向提示词**：
云架构图过度复杂，小字不可读，纯白底流程图，图标风格不统一，箭头混乱，遗漏 Redis/Milvus/OSS。
