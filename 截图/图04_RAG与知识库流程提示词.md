# 图04 RAG 与知识库流程提示词

## 用途

用于展示“RAG 事实增强”原理，建议放在“设计原理”章节。

## 生成提示词

请生成一张横版流程图，主题为“RAG 检索、知识增强与证据链生成”。画面要清楚展示从用户问题到检索、回复、洞察面板的过程，风格专业、简洁、适合放入作品设计说明书。

必须体现的流程：
- 用户输入问题
- 非遗知识库 `standardList.jsonl`
- 可选向量检索 Milvus
- 本地相似度回退
- 构造 Prompt
- 多 Agent 流式回答
- 消息 metadata 写入 `chat_message`
- 洞察接口 `/api/chat-rooms/{roomId}/insight`
- 知识摘要 / 关键证据 / 综合置信度 / 溯源信息
- 如果 metadata 缺少证据，则回退到 RAG 检索或最近群聊记录生成兜底证据

画面表达方式：
- 用箭头连接各模块
- 用编号步骤表示处理顺序
- 每个模块下方配简短中文说明
- 置信度节点建议标注“基于证据数量、RAG/搜索来源和多智能体一致性评估，展示值通常约 0.85-0.95”

视觉要求：
- 扁平化流程图
- 排版整齐
- 字体清晰
- 可以带少量图标，但不要抢占主体信息

负面约束：
- 不要画成复杂工程图
- 不要出现模糊的英文长句
- 不要太多装饰性光效
