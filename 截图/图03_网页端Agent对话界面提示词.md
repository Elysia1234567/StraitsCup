# 图03 网页端 Agent 对话界面提示词

## 用途

用于“设计原理”或“测试与运行效果”章节，展示当前已经实现的网页端 Agent 对话页面。图题建议写为“网页端 Agent 对话界面”，不要再写“多智能体群聊示意图”。

## 生成提示词

请根据最新 `网页端Agent对话.jpg` 的结构生成一张横版产品界面图，表现“多智能体群聊协作台”的真实页面。画面必须像已经上线的 Web 应用界面，不要像概念海报或旧版演示图。

画面内容建议包含：
- 顶部保留站点导航栏，左侧只保留圆形图标，不出现“同源”文字标识；导航项包含 Home、Agent、Models、Graph，Agent 为当前页高亮。
- 页面主标题为“多智能体群聊协作台”，副标题显示当前房间名称和时间，不要出现“图03”或“示意图”字样。
- 右上方的“文化问答”“知识增强”“流式回复”是固定文字状态按钮，不做可点击开关效果。
- 左侧为“群聊成员”面板，显示 1-6 个 Agent 头像、名称、简介和删除按钮；不要显示“工艺专家”“材料专家”等角色标签或标题。
- 左侧底部提供“邀请更多 Agent”或“添加 Agent”入口，表示可新增 Agent。
- 中间为真实聊天室：标题“群聊对话 · 非遗主题”，只保留右侧“新建聊天室”按钮；删除在线、设置等无关按钮。
- 对话消息只显示 Agent 名称和内容，不在消息中显示“材料专家”等角色标签；消息可体现多个 Agent 流式回答。
- 输入区包含文本输入、@ 所有人、上传图片、知识增强和发送按钮。
- 右侧为“知识增强与证据链”面板，包含“证据链 / 知识图谱 / 溯源信息”三个页签。
- 右侧证据链页签展示知识摘要、关键证据列表、综合置信度进度条，置信度建议在 0.85-0.95 之间，表现已经调高后的可信评估。
- 溯源信息区域应表现 RAG、本地知识库、联网搜索或群聊记录的来源，不要留空。
- 新建聊天室弹窗可以暗示历史聊天室选择，底部按钮必须完整可见，不要被遮挡。

表现重点：
- 每个 Agent 有不同头像和名称
- 对话流式感强，像实时输出
- 突出可增删 Agent、历史聊天室、WebSocket 流式回复、知识摘要、关键证据、综合置信度和溯源信息
- 整体与最新深蓝色 Agent 页面保持一致，不要使用旧版顶部“同源 OmniSource”品牌区

视觉要求：
- 界面清晰，层次分明
- 适合放入比赛说明书
- 颜色延续深蓝、青绿、少量紫色的科技文化风格

负面约束：
- 不要过度卡通化
- 不要无意义长文本
- 不要把输入框、按钮做得太小
- 不要让聊天内容遮挡界面结构
- 不要出现“图03 多智能体群聊示意图”
- 不要出现“同源”左上文字标识
- 不要出现“工艺专家”“材料专家”等角色标签
