## 2.2.3 系统关键技术与开发框架

本项目采用 B/S 架构作为主要实现方式，用户只需通过浏览器即可访问福建地图入口、Agent 聊天室、关系图谱和 AIGC 共创等功能，前端展示与后端业务逻辑相互分离，便于部署、维护和后续扩展。相比传统 C/S 模式，B/S 架构不需要在终端额外安装复杂客户端，能够更好地适应比赛展示、课堂演示和跨设备访问的需求，因此更适合作为本项目的主要技术形态。移动端则采用 UniApp 跨端方案，能够在保持统一业务逻辑的同时，适配手机浏览、对话和图鉴查看等场景，使项目在网页端和移动端之间形成一致的用户体验。对于需要高稳定性的现场演示，系统也支持将后端、数据库和缓存集中部署在同一台主机上运行，以降低环境复杂度；对于正式运行，则可采用前后端分离并结合数据库、缓存和对象存储的分布式部署方式。

在关键开发技术方面，后端以 Spring Boot 3.3.6 作为核心框架，承担 REST 接口、WebSocket 通信、业务编排与服务装配等职责；借助 Spring AI 1.0.0-M6 对接兼容模式下的大模型接口，完成多智能体对话、知识增强和生成式能力调用；通过 MyBatis 实现 SQL 映射与数据访问，配合 MySQL 完成业务数据持久化；通过 Redis 支撑会话状态、缓存与实时协同；通过 WebSocket 实现聊天室消息推送和流式回复。前端以 Vue 3 和 Vite 为基础，结合 Vue Router 完成页面路由组织，结合 ECharts、relation-graph、Three.js 与 OGL 实现地图展示、关系图谱和沉浸式可视化效果。移动端则采用 UniApp 与 Wot Design Uni，保证在手机端也能完成轻量化浏览与交互。项目还引入阿里云 OSS、DashScope SDK、Milvus SDK 等第三方能力，用于图片存储、大模型调用和向量检索增强，从而构成一套面向非遗数字生命共创场景的完整技术体系。

选择上述技术框架的原因，主要在于其成熟性、扩展性和生态完整性较强。Spring Boot 适合快速构建稳定后端服务，Vue 3 和 Vite 适合实现高交互前端页面，UniApp 则有利于复用内容逻辑并覆盖移动端场景；Redis、MySQL 和 Milvus 则分别满足缓存、结构化存储与语义检索需求，能够支撑项目在知识问答、聊天室协作和生成归档等环节中的实际运行。
