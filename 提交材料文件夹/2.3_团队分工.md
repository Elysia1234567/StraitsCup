## 2.3 团队分工与开发协作

同源（OmniSource）由福州大学软件工程学院 7 名同学组成的项目小组共同完成，指导教师为程烨老师。围绕“地图入口、智能体对话、知识增强、AIGC 共创、多端展示”五个核心环节，团队按照功能边界划分为 **后端组（3 人）**、**Agent 组（3 人）** 和 **前端组（1 人）** 三个工作组，整体采用前后端分离、模块化协作的方式推进开发，分工清晰、接口先行、每日同步。

### 二、后端组分工细化

后端组承担“后台不崩、数据不乱、接口能用”的工程底座，三位同学按主后端、AIGC 后端、数据后端的方向各有侧重。

**吕东剑（主后端，项目协调人）**

1. 搭建 Spring Boot 3.3.6 后端基础框架，统一项目结构、配置规范与日志规范。
2. 引入 Redis 消息队列与异步任务机制，解决多用户同时触发对话和生图时的并发与卡顿问题，保证比赛现场多评委同时操作的稳定性。
3. 负责所有核心 REST 接口的总调度，统一对接前端组与 Agent 组的请求，约定接口契约并维护 OpenAPI 文档。
4. 基于 Spring WebSocket 实现聊天室实时通信与流式输出，使智能体回答能够以逐字推送的方式呈现。
5. 承担整体代码管理与进度协调，主持每日同步会，跟进各组阻塞问题。

**陈尚斌（AIGC 与 RAG 后端）**

1. 基于 Spring AI 1.0.0-M6 对接兼容 OpenAI 协议的大模型接口（如通义千问），完成多智能体对话与生成式能力的统一调用层。
2. 搭建 RAG 知识库，将整理好的福建非遗文献、匠人访谈与历史资料导入向量存储；优先采用轻量化部署方案，并保留 Milvus 作为可选向量增强组件。
3. 实现文生图接口，封装寿山石雕、脱胎漆器、提线木偶、德化瓷、漳州木版年画等不同非遗风格的提示词模板。
4. 对接语音合成与识别能力，预置闽南语、福州话等地方音色参数，支撑“数字传承人方言讲解”功能。
5. 负责后端容器化与 Docker 部署脚本，保障联调与展示环境的一致性。

**卓俊炜（数据与基础接口）**

1. 整理福建九地市省级以上非遗名录，重点标注同源项目（如妈祖信俗、闽剧、木偶戏等），为每个项目补齐简介、代表作品与历史渊源等结构化字段。
2. 实现地图数据接口、搜索接口与二级页面数据接口，支撑前端鼠标悬停显示信息、点击展开下级地域以及搜索跳转等交互。
3. 完成用户对话记录与生成作品的轻量化存储设计，保障历史聊天室、消息流和图像归档可被持续读取。
4. 配合主后端与 AIGC 后端编写公共接口、参与联调测试与缺陷修复，并维护后端 API 文档与数据字典。

### 三、Agent 组分工细化

Agent 组聚焦“AI 有脑子、有样子、会聊天”这一核心创新点，三位同学分别负责多智能体协作、形象人设与生成式视觉表达。

**林怡杰（多智能体主负责人）**

1. 搭建多智能体协作基础框架，定义“历史学家、老匠人、年轻游客”等核心角色，明确每个角色的知识边界与发言节奏。
2. 编写角色 Prompt，使各 Agent 能基于 RAG 知识库围绕同一主题（如妈祖信俗、闽剧唱腔、木偶戏流派）展开自主讨论。
3. 实现“用户介入群聊”机制：用户可随时插话，相关 Agent 会根据主题匹配进行针对性回应。
4. 优化对话编排逻辑，结合证据链与置信度展示，避免模型胡编乱造，确保所有回答均可溯源到知识库。

**陈禹帆（数字传承人形象与人设）**

1. 设计地域特色 AI 形象模板，覆盖福州寿山风、泉州木偶风、德化瓷韵风、漳州年画风等不同视觉原型，并支持拟人化风格扩展。
2. 为每个非遗项目编写专属人设 Prompt，使智能体的性格与表达风格与项目气质相匹配（如蜀绣温润细腻、皮影活泼诙谐）。
3. 对接后端语音接口，实现“数字传承人以方言讲解非遗”的能力，并提前调校各地口音参数。
4. 完成形象展示界面与基础动效（眨眼、口型、淡入淡出），让 AI 形象能够在对话页面中“活起来”。

**杨文渊（生图与 3D 辅助）**

1. 持续打磨非遗风格生图提示词，保证剪纸、水墨、皮影、彩塑等风格的统一性与辨识度（如剪纸风强调镂空与红色主调）。
2. 实现“对话中生图”能力：用户在聊天中表达视觉诉求时，系统自动生成图片并作为消息回流到当前聊天室。
3. 基于 Three.js 搭建 Web3D 虚拟展馆，将生成的非遗作品陈列于沉浸式展厅中。
4. 准备答辩演示用的生图案例，针对生成耗时进行预热与缓存优化，争取常见提示词在 3 秒左右出图。

### 四、前端组分工细化

**杨欣潼（Web 前端主负责）**

1. 搭建 Vue 3 + Vite 工程基础，按页面与组件层级组织代码，统一引入 UI 组件库与样式规范。
2. 实现首页地图界面：使用 ECharts 渲染福建地图，完成鼠标悬停信息提示、点击下钻到地市非遗项目、搜索框模糊匹配跳转等核心交互。
3. 实现非遗详情二级页面，按地域展示项目卡片，配套图片、简介与跳转入口。
4. 实现“豆包风格”的 Agent 对话界面：左侧聊天室列表、中部对话流、右侧知识增强与证据链，支持流式输出、图像消息和形象展示。
5. 设计生成与切换动效，包括生图加载动画、AI 形象淡入、3D 展厅旋转等效果，提升答辩观感。
6. 适配答辩现场的投影仪分辨率，针对大屏环境调整字号、留白与对比度，确保界面在大屏幕下清晰可读。

### 五、协作机制

为保证三个工作组在功能上互相衔接、在进度上并行推进，团队约定如下协作机制：

1. **接口先行**：后端组在动手实现前先完成接口约定，前端组与 Agent 组依据接口文档并行开发，减少串行等待。
2. **每日同步**：每日由项目协调人吕东剑主持 5 分钟站会，三组各自同步进展、阻塞和当日目标，问题当天暴露、当天定位。
3. **统一仓库**：后端、前端、移动端与 Agent 配置统一纳入项目仓库管理，按分支隔离功能开发，重要节点合并后做联调验收。
4. **联调与演练**：在关键节点（如 RAG 接入、生图接入、群聊上线、3D 展示上线）安排专项联调，并模拟答辩现场流程进行端到端演练，确保从地图入口到 AIGC 共创的完整链路可一次性跑通。

通过上述分工与协作机制，团队在保证功能完整度的同时，也兼顾了系统稳定性与答辩现场的演示效果，使每位成员的工作都能在同源（OmniSource）这一作品中清晰落地。
