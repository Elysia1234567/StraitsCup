## 2.2.2 系统运行环境

本项目采用 B/S 架构与分布式服务协作方式运行，整体由前端展示端、后端业务服务、数据库与缓存服务、对象存储服务以及第三方 AI 服务共同组成。比赛答辩或本地演示时，也可以将后端、数据库与缓存集中部署在同一台主机上，前端通过浏览器访问，以便简化现场环境；正式运行时，则建议按照分布式方式部署各组件，以提高系统稳定性与扩展性。

### 一、系统运行模式

1. **前端展示层**：运行在浏览器或移动端 WebView 中，负责页面呈现、地图浏览、聊天室交互、图谱展示和图片查看。
2. **业务服务层**：运行 Spring Boot 后端服务，负责用户请求处理、聊天室管理、知识检索、AIGC 调度、WebSocket 推送等核心业务。
3. **数据支撑层**：运行 MySQL、Redis 和可选 Milvus，用于业务数据持久化、缓存、会话状态和向量检索增强。
4. **外部能力层**：通过网络访问阿里云 OSS、DashScope、千问兼容接口、Tavily 等第三方平台，完成图片存储、大模型调用和扩展检索。

### 二、系统部署要求

#### 1. 前端设备部署要求

- 设备类型：普通个人计算机、笔记本电脑、平板或智能手机。
- 操作系统：Windows 11 24H2、macOS、Android、iOS 等主流系统均可。
- 浏览器要求：Chrome、Edge、Safari 等支持现代 JavaScript 和 WebSocket 的浏览器，建议使用较新版本。
- 运行软件：无需安装额外服务器软件，只需通过浏览器访问前端页面即可。

#### 2. 后端服务设备部署要求

- 设备类型：x86_64 架构服务器、云主机或本地开发机。
- 操作系统：Windows 11 24H2、Windows Server、Linux 均可。
- 运行环境：JDK 17。
- 服务版本：Spring Boot 3.3.6，Spring AI 1.0.0-M6，MyBatis 3.0.3，Spring Data Redis 3.3.6，Spring WebSocket 3.3.6。
- 其他要求：具备外网访问能力，以便调用第三方 AI 接口、对象存储和检索服务。

#### 3. 数据库与缓存设备部署要求

- MySQL：建议使用 MySQL 8.0 及以上版本，用于存放聊天室、消息、智能体配置和生成任务等结构化数据。
- Redis：建议使用 Redis 7.0 及以上版本，用于缓存、会话与实时状态管理。
- Milvus：如启用向量增强能力，建议部署 Milvus 2.3.x 及以上版本；若不启用，可直接使用本地知识库完成基础检索。

#### 4. 对象存储与外部服务要求

- 阿里云 OSS：用于存放生成图片、资源文件和用户上传内容，部署端仅需具备访问 OSS 服务的网络权限。
- DashScope / 千问接口：用于大模型对话、图像生成和语音识别，服务端需配置有效 API Key。
- Tavily：用于联网补充检索，服务端需具备对应 API Key 与外网连通能力。

### 三、系统通信网络说明

系统通信采用“客户端—后端服务—数据与外部能力服务”的多段式连接方式。

1. **前端到后端**：浏览器或移动端通过 HTTP/HTTPS 请求访问 REST 接口，通过 WebSocket 与后端建立长连接，实现聊天室消息、流式回复和实时状态同步。
2. **后端到数据库**：后端通过 MyBatis/JDBC 访问 MySQL，通过 Spring Data Redis 访问 Redis，通过 Milvus SDK 访问 Milvus。
3. **后端到对象存储**：后端通过阿里云 OSS SDK 上传图片、生成结果和附件，并在数据库中保存资源地址。
4. **后端到第三方 AI 服务**：后端通过 Spring AI 和 DashScope SDK 调用千问兼容接口、语音识别接口及相关生成服务。
5. **前端展示联动**：前端在接收到后端推送的数据后，分别驱动地图、图谱、3D 展示和对话界面完成渲染。

网络连接示意为：

用户终端 → 前端页面 → 后端业务服务 → MySQL / Redis / Milvus / OSS / 第三方 AI 服务

其中，前端与后端之间主要依赖局域网或公网 HTTP/WebSocket 通信；后端与数据库、缓存和对象存储之间主要依赖内网或云网络连接；第三方 AI 服务通过公网 API 调用实现。对于答辩现场，若采用单机演示模式，则可将后端、数据库和缓存部署在同一台机器上，前端通过本机地址访问，从而降低环境复杂度。

### 四、各设备上的系统软件配置

| 设备类型 | 部署软件 | 版本要求 | 作用 |
|---|---|---:|---|
| 用户终端 | Chrome / Edge / Safari 等浏览器 | 建议较新稳定版 | 访问前端页面、执行 WebSocket 交互、查看图谱与图像 |
| 前端开发机 | Node.js、Vite、Vue 3 相关依赖 | Node.js 较新 LTS 版本，Vite 8.0.4，Vue 3.5.32 | 前端本地开发、调试与构建 |
| 后端服务机 | JDK 17、Spring Boot、MyBatis、Spring AI、Spring WebSocket | JDK 17，Spring Boot 3.3.6，Spring AI 1.0.0-M6 | 提供业务接口、实时通信和 AI 调度 |
| 数据库服务器 | MySQL | 8.0 及以上 | 存储核心业务数据 |
| 缓存服务器 | Redis | 7.0 及以上 | 缓存、会话与状态管理 |
| 向量检索服务器 | Milvus | 2.3.x 及以上 | 可选语义检索与知识召回增强 |
| 对象存储服务 | 阿里云 OSS | 服务端托管版本 | 存储图片与资源文件 |
| AI 能力服务 | DashScope / 千问兼容接口 | 以服务端最新可用版本为准 | 大模型对话、图像生成、语音识别 |

### 五、运行方式说明

- **集中式部署**：适用于现场演示和调试，将后端、MySQL、Redis 以及可选 Milvus 部署在同一台主机上，前端通过本机或局域网地址访问。
- **分布式部署**：适用于正式运行，将前端、业务服务、数据库、缓存、对象存储和 AI 服务分别部署在不同节点或云服务中，通过网络协同工作。

综合来看，本项目以分布式架构为主，集中式部署为辅，既满足正式运行的可扩展性，也便于比赛现场快速部署与演示。
