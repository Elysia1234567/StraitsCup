## 2.2.1 系统开发平台（含开源/第三方工具）

本项目采用“后端 Spring Boot + 前端 Vue 3 + 移动端 UniApp + 数据库与中间件 + 第三方 AI/存储服务”的技术路线，开发、联调与展示环境均围绕 Windows 个人计算机与常见云服务部署方式构建。项目主要使用的开发平台、开源框架和第三方工具如下。

### 一、操作系统与基础开发环境

- 操作系统：Windows 11 24H2（微软 Microsoft 出品）。
- Java 开发环境：JDK 17。
- 构建工具：Maven 3.x。
- 前端构建环境：Node.js 运行环境，配合 Vite 完成前端工程启动与打包。

| Spring AI | Spring | 1.0.0-M6 | 对接大模型能力，完成兼容 OpenAI 接口的聊天与生成调用 |

### 五、数据库与数据存储平台

| 名称 | 厂商/组织 | 版本号 | 作用 |
|---|---|---:|---|
| MySQL | Oracle | 部署环境决定 | 存储聊天室、消息、智能体、任务等核心业务数据 |
| Redis | Redis Ltd. / 开源社区 | 部署环境决定 | 缓存、会话、状态与实时消息辅助存储 |
| Milvus | Zilliz / 开源社区 | 可选部署 | 向量检索与知识召回增强 |
| 阿里云 OSS | 阿里云 | 服务端版本随平台 | 存储图片、生成结果与资源文件 |

其中，MySQL 负责结构化业务数据持久化，Redis 负责实时状态与短期数据缓存，Milvus 仅在开启 RAG 向量增强时使用，阿里云 OSS 则用于保存图片和生成内容。多种数据存储之间通过业务层统一编排：业务主数据写入 MySQL，短期会话与热点状态放入 Redis，语义知识通过本地知识库和可选 Milvus 关联，图片与附件统一存入 OSS，并在数据库中保存访问地址与元数据。

### 六、开源平台与第三方工具出处及已实现功能

1. Spring Boot、Spring AI、MyBatis、Vue 3、Vite、ECharts、Three.js、OGL、relation-graph 等均来自各自官方开源社区或项目主页，项目中主要用于后端服务搭建、前端页面开发、地图与图谱展示、3D/WebGL 展示等功能实现。
2. DashScope SDK、Alibaba OSS SDK、MySQL Connector/J、Milvus SDK 等均来自对应官方 SDK 或开源仓库，项目中分别用于大模型调用、语音识别接入、对象存储、数据库连接与向量检索增强。
3. 本项目未直接拷贝第三方项目的业务源码，主要通过官方 SDK、组件库和开源框架完成二次集成与功能实现。

### 七、硬件设备与运行环境

本项目未绑定特定的专用硬件设备，开发、调试与展示均可在通用 x86_64 架构个人计算机或服务器上完成。当前工程在普通 Windows 桌面环境下进行开发与联调，后端可部署于满足 JDK 17、MySQL、Redis 与网络访问条件的服务器，前端可运行于常见浏览器环境。项目未强制要求 GPU、深度学习工作站或其他特殊硬件。

如需正式提交材料时写入更具体的设备型号，可在最终答辩机或部署机确认后补充为：

- 开发设备：Windows 桌面个人计算机，型号与厂商以实际使用设备为准。
- 部署设备：通用 x86_64 服务器或云主机，型号与厂商以实际部署资源为准。
