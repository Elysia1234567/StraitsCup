## 一、作品简介

《同源》是一套面向福建非遗文化数字传播的智能交互平台。作品以“非遗数字生命”为核心概念，将福州、厦门、泉州、漳州、莆田、南平、三明、龙岩、宁德等地的代表性非遗内容转化为可对话、可协作、可生成图像的文化 Agent。用户既可以通过福建地图与环形画廊浏览地域文化，也可以进入多智能体聊天室，与不同器灵 Agent 进行实时交流，还可以让指定 Agent 根据自身文化原型生成图像作品。

项目已形成前端、后端、知识库、实时通信、AIGC 与对象存储的完整闭环，具备真实部署和现场演示基础。

## 二、设计原理

作品采用“表现层 - 业务编排层 - 智能能力层 - 数据支撑层”的分层设计。

前端基于 Vue 3 组织页面和组件，通过 `useMultiAgentChat` 组合式逻辑管理房间、消息、WebSocket 连接和流式分片；首页以福建地图联动非遗画廊，聊天页提供智能体库、聊天室、历史消息和 AI 生图面板。后端基于 Spring Boot 3，将接口入口放在 Controller，核心业务放在 Service 与 ServiceImpl，数据访问由 MyBatis Mapper 承担，实体和 DTO 分层清晰。

智能回答链路中，`AgentOrchestrator` 根据 Agent 知识范围与用户问题计算相关度，并利用 Redis 记录发言冷却，避免重复抢答；`RagServiceImpl` 将本地 JSONL 非遗资料导入 Milvus，使用千问 Embedding 构建向量检索，同时保留本地字符相似度兜底；`AgentPromptBuilder` 将 Agent 人设、RAG 片段、联网搜索资料、输出约束和用户问题组装为 Prompt，再由 Spring AI 调用兼容 OpenAI 协议的大模型。生成过程通过 WebSocket 的 `AGENT_START`、`AGENT_CHUNK`、`AGENT_END` 等消息类型实时推送，前端按 `streamId` 增量渲染。

图像生成链路中，用户选择房间内某个 Agent 并输入画面描述；后端根据 Agent 名称、编码、性格和参考图 URL 构造生图 Prompt，调用图像模型生成结果，再下载并上传至阿里云 OSS，最终以图片消息形式写入聊天室历史。

## 四、创新点

1. **非遗器灵 Agent 化**  
   作品没有停留在百科问答，而是把非遗对象抽象成“器灵”角色。每个 Agent 具备唯一编码、头像、人格摘要、知识范围、语言风格、约束条件、生成参数等配置，既能保持文化特征，又便于工程扩展。

2. **RAG 事实约束与沉浸式表达兼容**  
   `AgentPromptBuilder` 明确规定资料优先级：Agent 人设约束高于 RAG，RAG 高于联网搜索，搜索高于模型自身知识。对于特殊沉浸角色，系统还支持 `NO_RAG` 模式，让角色只谈记忆、情绪和自我叙述，形成“知识型 Agent”和“叙事型 Agent”的双轨机制。

3. **多智能体编排机制**  
   `AgentOrchestrator` 不是让所有 Agent 无差别回答，而是基于知识范围匹配、Redis 冷却时间和最多 3 个响应者限制进行选择；用户输入“你们”“大家”等词时可触发全员响应；自由讨论时可触发接力发言，增强真实群聊感。

4. **RAG 检索双保险**  
   系统优先使用 Milvus + Embedding 做语义检索；当 Milvus、Embedding 或外部服务不可用时，自动回退到本地检索，通过标题、内容、类别、地区、级别、代表作品等字段进行评分，保证演示时仍可返回资料。

5. **生成式文化共创闭环**  
   AIGC 不只是附加功能，而是与 Agent 人设、参考图、聊天室历史和 OSS 归档连接。生成结果会成为对话中的一条图片消息，用户可以继续围绕它讨论，实现“问答 - 创作 - 再阐释”的循环。

6. **工程模式清晰**  
   项目体现了 MVC 分层、依赖注入、接口与实现分离、构建器模式、注册表模式、编排器模式、策略式 Prompt 分支、观察者式 WebSocket 广播、异步任务与缓存状态管理等多种工程设计思路。

## 五、实用点

- 可用于非遗展馆、校园文化节、博物馆教育、研学活动和数字文旅展示。
- 支持用户低门槛提问，把专业非遗资料转化为更易理解的角色化表达。
- 支持多 Agent 群聊，可模拟历史讲解者、工艺讲述者、审美创作者等不同视角。
- 支持图像生成和归档，可辅助宣传海报、展览视觉草图和文化衍生品创意。
- 后端具备数据库表、接口文档、异常处理、测试类和可配置模型服务，便于继续迭代。

## 六、关键设计模式与工程亮点

| 类型 | 代码体现 | 价值 |
|---|---|---|
| MVC 分层 | `controller / service / mapper / entity / dto` | 降低耦合，便于定位和扩展 |
| 依赖注入 | Spring 构造器注入、`@Service`、`@Component` | 让服务组合清晰，可测试性更好 |
| 构建器模式 | `AgentDefinition.builder()`、`WebSocketMessage.builder()`、响应 DTO | 复杂对象创建更清晰 |
| 注册表模式 | `FujianCulturalAgentRegistry` 汇总九地市定义 | 便于批量扩展文化 Agent |
| 编排器模式 | `AgentOrchestrator` 选择发言 Agent | 将协同规则从聊天服务中解耦 |
| 策略式分支 | RAG Prompt 与 `NO_RAG` 沉浸 Prompt | 同时支持知识问答和角色叙事 |
| 观察者 / 发布订阅 | `WebSocketSessionManager.broadcastToRoom` | 支撑多人房间实时更新 |
| 兜底设计 | Milvus 失败回退本地检索 | 提升演示和运行稳定性 |
| 异步任务 | `@Async processTask`、Redis 任务状态 | 适合处理耗时图像生成 |

## 七、总结

《同源》把非遗数字化从“资料展示”推进到“智能生命交互”。作品在文化表达上突出地域同源、器物拟人和用户共创；在技术实现上完成了 Vue 前端、Spring Boot 后端、RAG 知识库、多智能体编排、WebSocket 流式通信、AIGC 生图和 OSS 归档的闭环。后续可进一步扩展知识规模、增强事实审核、完善账号权限和多端适配，使其成为可持续迭代的非遗数字传播平台。
