## 一、项目概述

《同源》面向福建九地市非遗文化传播场景，围绕“非遗数字生命”这一核心概念，将分散的文化条目组织为可对话、可协作、可生成、可展示、可沉淀的文化 Agent。用户可以从福建地图进入文化场景，也可以进入多智能体聊天室，在同一主题下持续追问、切换历史房间、增删成员，并让指定 Agent 依据自身文化原型生成视觉内容。

与传统静态展示不同，本项目强调“文化内容进入会话空间”。系统不只回答问题，还会自动整理知识摘要、关键证据、综合置信度和来源线索，使答案具备可读性、可追溯性和可展示性。用户在网页端、移动端和沉浸式页面之间切换时，能够看到统一的文化识别风格与连续的交互逻辑。

## 二、总体设计

本项目采用前后端分离、分层协作的整体思路，将界面呈现、实时通信、知识检索、生成调度和数据沉淀组织为一个完整闭环。

在用户体验层，网页端负责地图入口、非遗画廊、Agent 聊天室、知识面板、关系图谱和 3D 展示；移动端则提供首页、对话页、智能体页、图鉴页和接口实验页，便于在不同设备上演示统一能力。界面设计以深蓝、青绿和少量金色为主，强调文化感和科技感的平衡。

在实时交互层，系统通过实时消息机制承载用户提问、智能体流式回复、图片结果、系统提示和进度反馈。多个 Agent 可以并发生成内容，前端通过独立的消息标识分别拼接，避免串流和内容错位。

在业务编排层，系统负责聊天室管理、成员增删、历史房间恢复、知识检索、生成任务和消息持久化。用户在一个房间里既可以查看历史，也可以继续扩展成员，形成真正可运营的文化圆桌。

在智能能力层，系统集成文本问答、图像理解、图像生成、联网补充和语音识别能力，并通过本地知识库与可选向量增强共同支撑事实性回答。对于特殊叙事角色，系统保留更自由的表达方式，避免所有角色都被同一种回答风格淹没。

在数据支撑层，系统以关系型数据库承载 Agent、主题、房间、消息和生成任务等核心数据，以缓存机制支撑实时状态和任务进度，以对象存储保存生成结果，以向量检索增强知识召回能力。

## 三、数据组织与知识支撑

项目的数据组织围绕五类对象展开：文化 Agent、非遗主题、聊天室、消息和生成任务。这样的设计既满足普通业务流程，也服务于文化知识沉淀和后续扩展。

文化 Agent 保存角色名称、头像、人设、知识范围、语言风格和生成约束。非遗主题保存主题名称、地区、分类、摘要和封面信息，作为展陈入口和问答入口的统一索引。聊天室保存当前主题、成员数量、消息数量和状态信息，支持用户创建、恢复和管理不同的文化讨论空间。消息则统一保存文本、图片、系统提示和流式内容，让文化交流具备历史轨迹。生成任务记录图片创作过程中的提示词、风格、状态、结果地址和错误信息，便于追踪和归档。

在知识支撑层面，系统先保证本地知识库可用，再根据语义相关度进行增强召回；当外部向量环境不可用时，会自动回退到本地相似度检索，保证现场展示不会因为单点依赖而中断。检索结果并不会停留在后台，而是进一步汇总进聊天室洞察面板，形成知识摘要、关键证据、综合置信度和溯源信息。

这些洞察内容的来源并不单一，而是综合最近提问、最新回复、知识检索结果、联网补充结果和对话上下文共同生成，因此用户看到的不是空白面板，而是一条连续的证据链。

## 四、多智能体协作与流式对话

项目的核心体验不是单轮问答，而是围绕同一文化主题展开的多智能体协作。不同 Agent 拥有不同的性格、表达方式和知识边界，在同一个房间里形成互补回答，既有专业解释，也有叙事表达。

当用户提问后，系统会根据问题内容、当前房间成员和发言节奏选择适合回应的智能体，必要时让多个 Agent 共同回答。系统会控制连续抢答与重复输出，让群聊更像自然交流而不是机械轮流发言。

在网页端，左侧显示成员列表，支持继续添加或删除 Agent；中间是主聊天区，展示真实的流式回复；右侧是知识增强与证据链面板，展示摘要、证据、置信度和来源。界面上方保留站点导航和固定状态文案，下方保留历史聊天室入口与新建聊天室功能，使用户可以在一个页面内完成切换、讨论和整理。

流式回复到达时，前端会以分片形式逐步渲染，形成打字机般的实时感。这样做的好处是，评审不仅能看到最终结论，还能看到内容生成过程，感受到系统的“思考轨迹”。

## 五、AIGC 生图与入聊机制

项目并未把图像生成当作独立功能，而是把它融入对话上下文。当用户选择房间中的某个 Agent 并输入画面描述后，系统会结合该角色的人设、头像、文化原型和当前房间语境构造更稳定的生成提示，再将结果以图片消息的形式写入聊天室。

生成过程包含任务创建、状态推进、结果保存和归档四个阶段。生成中的图片可以在界面上看到进度，成功后图片会被上传到对象存储，并同步回聊天室历史。这样，图像不再是一次性的输出，而是可以继续被讨论、补充和传播的文化材料。

这种机制让“回答”与“创作”发生了连锁反应：先由智能体解释，再由用户触发视觉表达，最后把结果重新放回对话空间，形成二次讨论和再阐释。

## 六、前端与移动端展示

网页端强调实时交互与沉浸展示。首页通过福建地图和非遗画廊建立入口，聊天页承载多智能体协作，关系图谱页呈现文化关联，3D 页面展示沉浸式视觉资产。移动端则提供轻量化原型，让同一套内容在手机上也能完成浏览、对话和展示。

网页端与移动端并不是两套割裂的系统，而是同一体验逻辑在不同屏幕上的统一呈现。用户在网页端进入深度讨论，在移动端继续查看或检索，整体视觉与信息结构保持一致，有利于比赛现场和日常演示。

## 七、创新点与运行闭环

本项目的创新并不只体现在“接入了大模型”，而在于把文化对象、知识检索、群聊协作、证据链、生成内容和展示方式连成一体。

一是非遗对象被角色化。非遗不再只是静态条目，而是可以被召唤、对话和持续追问的文化主体。每个 Agent 都有稳定的人设、语气和知识边界，便于形成记忆感和辨识度。

二是群聊协作可控。系统不是让所有角色无差别输出，而是通过主题匹配、发言节奏和房间状态，让对话既丰富又不拥挤。

三是知识链路可解释。摘要、证据、置信度和来源线索让回答不只是“像对的”，而是能说明为什么这样回答。

四是生成内容可回流。图像生成不是结尾，而是新的对话起点，能够把文化创作、视觉表达和知识传播串成闭环。

五是多端展示一致。Web、移动端和沉浸式页面共享统一的文化语言，使评审能在不同屏幕上快速建立对项目的整体印象。

## 八、运行与总结

从实际运行效果看，项目已经形成“地图入口 - 成员管理 - 多智能体对话 - 知识洞察 - 图像生成 - 归档展示”的完整链路。即使向量增强环境不可用，系统依然可以依靠本地知识库和相似度检索工作，保证现场演示稳定。

当用户打开页面后，可以先从城市入口进入，再切换历史聊天室，补充新的 Agent，查看右侧证据链，必要时上传图片或触发生图，最后在图谱和 3D 页面中延展展示。整个过程中，系统不仅完成信息传递，也完成文化叙事。

这说明《同源》已经从“非遗展示工具”迈向“文化 Agent 共创平台”。后续如果继续扩展内容规模、丰富角色体系、增强审核机制和完善多端交互，作品还可以进一步演化为适用于展馆、课堂和文旅场景的长期平台。
