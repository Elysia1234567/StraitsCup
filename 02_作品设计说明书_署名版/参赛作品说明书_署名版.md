## 一、作品简介

同源（OmniSource）是一个面向福建九地市非遗文化的数字生命共创平台。项目将非遗项目组织为具有人设、知识边界、协作能力和生成能力的文化 Agent，用户可以通过福建地图进入非遗内容，在聊天室中与多个“数字生命”对话，并围绕非遗原型生成视觉作品。

项目不是单一聊天机器人，而是把多智能体问答、RAG 检索、本地知识回退、AIGC 生图、WebSocket 流式交互、OSS 归档、WebGL 展示和移动端原型串成完整闭环，形成“可进入、可对话、可协作、可生成、可沉浸展示”的非遗数字化体验。

## 二、设计原理

本项目的设计原理，是将非遗文化资源从静态内容组织为可交互、可协同、可生成的数字化文化资产。每个文化智能体都对应一类非遗主题，具备角色身份、知识边界、表达风格与文化语境，因而能够从“内容检索”升级为“文化叙事”。

在知识组织层面，系统以本地 JSONL 知识库、向量知识库和语义回退机制共同构成检索基础，兼顾准确性、连续性与容错性。当前后端默认先保证 `Util/standardList.jsonl` 可用，Milvus 作为可选增强，仅在显式开启 `RAG_MILVUS_ENABLED=true` 时参与同步。

在协同表达层面，系统采用多智能体分工机制，让不同角色围绕同一文化主题形成互补性回答，体现知识专家、讲述者、引导者和生成者的分层协作。回答不再是单一视角，而是具备文化背景、工艺细节、地域特征与传播价值的复合表达。

在生成层面，系统将智能体人格、参考原型与用户意图共同纳入生成语境，使视觉结果具备一致的文化识别度。生成内容进入会话后，又通过对象存储、消息持久化和实时推送形成可追溯的展示链路，保证“生成即展示、展示即归档”。

## 三、技术架构与技术路线

前端采用 Vue 3 + Vite + Tailwind CSS + ECharts + relation-graph + OGL + Three.js + GSAP，完成福建地图入口、非遗画廊、Agent 聊天室、AIGC 生图面板、关系图谱、无限 WebGL 画廊和 3D 模型展示。移动端采用 UniApp，提供首页、对话、智能体、图鉴和接口实验室页面。

后端采用 Spring Boot 3.3 + Spring AI + MyBatis + MySQL + Redis + WebSocket + OSS，接入通义千问兼容模型、Embedding、图像生成、图像理解、Tavily 搜索和 DashScope Fun-ASR，构成完整的技术闭环。Milvus 用于向量增强，但不会阻塞应用启动。

技术路线可概括为：

非遗知识库 -> RAG 检索 -> Agent 编排 -> WebSocket 流式对话 -> AIGC 生图 -> OSS 归档 -> 前端实时呈现。

其中，前端负责信息呈现与交互，后端负责知识检索、Agent 调度、生成任务和消息持久化，二者通过 REST、SSE 与 WebSocket 共同支撑实时体验。系统技术画像也可通过 `/api/system-profile` 直接输出，便于答辩展示。

## 五、创新点

第一，将非遗项目从单点展示升级为文化智能体体系，让文化对象具备人格、语境、边界与传播方式，形成可持续的数字叙事单元。

第二，构建多智能体协同机制，使文化知识从单一输出转变为多角色共创，兼顾专业解释、叙事表达与互动体验。

第三，融合结构化数据库、向量知识库与本地回退策略，使文化内容同时具备检索效率、知识可信度与现场演示稳定性。

第四，将生成式视觉内容纳入会话空间，使文化传播不止停留在阅读和问答，还能延伸到形象化表达和沉浸式呈现。

第五，采用多平台一致化呈现策略，在 Web、移动端与沉浸式展示空间之间保持统一的视觉识别与交互逻辑。

## 六、实用点

项目适合比赛展示，也适合后续继续扩展为非遗传播平台。它能够用于文化科普、互动讲解、数字展陈、移动浏览和课堂演示。

系统已经具备较强的工程可落地性：具备数据库持久化、实时通信、知识检索、图像生成、对象存储与多端展示能力，能够满足展示、讲解、演示与后续扩展等多种应用场景。对于评审而言，既能看到创意，也能看到实现深度。

## 七、测试与运行效果

当前项目已经形成“地图入口 - 智能体协作 - 文化问答 - 视觉生成 - 归档展示”的完整体验闭环。前端界面可呈现地域入口、智能体成员、聊天对话、生成图片、关系图谱和三维展示等多个层次，适合比赛现场的连续演示。

在运行效果上，系统能够稳定支撑多平台访问、实时交互、知识增强问答与生成式内容沉淀。即使 Milvus 未开启，系统依然可通过本地 JSONL 和相似度检索工作，保证展示不被单点依赖打断。

## 八、总结与后续改进方向

同源（OmniSource）已经完成从“非遗展示”到“文化 Agent 共创”的转变，具备多智能体、RAG、AIGC、实时通信和多端展示等核心能力。

后续可以继续完善的方向包括：补充正式登录鉴权、进一步提高检索数据质量、扩充更多非遗条目、优化移动端交互、补强匿名材料脱敏检查，并继续打磨答辩展示素材。
